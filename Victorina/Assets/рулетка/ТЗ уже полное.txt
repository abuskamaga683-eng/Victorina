Здравствуйте, впервый раз обращаюсь, не очень много знаю о данном сайте. Требуется создать игру викторину на юнити (2д). 

Название игры: Опасности и Безопасность.

Игра будет использоваться для дипломного проекта.

1. Функциональные требования

1.1. Приложение должно обеспечивать полный цикл прохождения викторины: от запуска до вывода итоговых результатов.

1.2. Стартовое окно должно предоставлять выбор режима работы:

запуск игры;

переход в административный режим;

выход из приложения.

1.3. Перед началом игры должен быть предусмотрен ввод названий команд (участников). Количество команд – от 2 до 4.

1.4. Запуск игрового процесса осуществляется через механизм случайного выбора вопросов (рулетка категорий).

1.5. Игра должна выводить вопросы по различным категориям безопасности, загружаемые из базы данных.

1.6. Ответы даются участниками устно, чтобы выбрать команду давшую правильный ответ требуется лишь выбрать в программе самостоятельно кто дал правильный ответ и продолжить дальше.

1.7. В процессе игры должны отображаться текущие результаты (счет команд).

1.8. По завершении игры должен определяться и отображаться победитель. 

1.9. Административный режим должен позволять редактировать базу вопросов (добавление, изменение, удаление вопросов и категорий).

1.10. Игровой процесс должен исключать повторение уже использованных в текущей сессии вопросов для обеспечения объективности.

2. Нефункциональные требования

2.1. Интерфейс должен быть простым и интуитивно понятным для пользователей разного уровня подготовки.

2.2. Система должна обеспечивать высокую скорость отклика (отсутствие заметных задержек при взаимодействии).

2.3. Приложение должно работать стабильно, без зависаний и критических ошибок.

2.4. Все элементы интерфейса должны корректно отображаться на целевых разрешениях экрана.

2.5. Данные в базе вопросов должны сохраняться при любых корректных действиях пользователя (без потерь).

2.6. Навигация по этапам викторины должна быть логичной и не требовать дополнительных инструкций.

2.7. Приложение должно быть автономным – полностью функционировать без подключения к сети Интернет, без скачивания дополнительных файлов и без установки внешних СУБД. Все данные (вопросы, ответы, результаты) хранятся локально во встроенной базе SQLite (либо любой другой по вашему усмотрению и удобству, но если есть возможность, то уточнить это).

Выбор вопросов происходит в качестве прокрута рулетки.

Также по возможности хотелось бы видеть прототипы разработки (много не требуется, хотя бы по 1 скриншоту этапов разработки для дипломного проекта).
